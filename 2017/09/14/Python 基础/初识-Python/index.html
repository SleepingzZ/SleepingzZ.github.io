初识 Python

Author: Celeste° Date: 2017-September-14  13:59:54 Category: Python

一、计算机组成原理

1. 什么是编程？

  1. 编程本质就是一门语言，语言是一种事物与另一种事物沟通的工具，所以编程语言也是一种事物与另一种事物沟通的工具

  2. 人 —-> 编程语言 —-> 计算机

    • 在 IT 领域，人通过编程语言与计算机沟通，以此来奴役计算机

  3. 两个步骤：

    • 理清逻辑思路
    • 用计算机能听懂的语言（即编程语言）来命令计算机工作

  4. 程序由功能（代码）和数据组成

  5. 软件是存放程序的文件，用来控制计算机，是计算机的灵魂

2. 计算机的硬件组成与运行原理

  • 计算机的核心硬件组成都是为了模仿人的某一器官或者功能

  1. 控制器 + 运算器 —-> CPU —————————————————————————- 大脑

    • 控制器：计算机的指挥系统，负责控制所有其他硬件的运行 —————————— 大脑
    • 运算器：负责数学运算与逻辑运算 ————————————————————— 大脑

  2. 存储器：内存与外存存储的分别为高低电频与磁频，统一用 0、1 表示

    • 内存：存的是电信号，断电数据丢失 ——————————————— 大脑的记忆功能

      • 特点：存取速度快，但只能临时存放数据

    • 外存（磁盘）：存放的是磁信号，断电数据依然存在 ———————————— 笔记本

      • 特点：存取速度慢，但可以永久存放数据

  3. 输入设备：键盘、鼠标

  4. 输出设备：显示器、音响、打印机

内存
  • RAM：内存中大部分为 RAM 数据，即可读可写的数据，且为易失性存储，断电后数据全部消失
  • ROM：内存中有少量 ROM（Read Only Memory）数据，即只读数据，无法更改，且为非易失性存储，断电后内容不会丢失
  • EEPROM 和闪存：与 ROM 一样，也是非易失性的，但可以擦除和重写，不过重写时花费的时间比写入 RAM 要多
  • CMOS：为易失性存储，计算机中一般用来保持当前时间和日期以及保存配置参数。CMOS 耗电非常少，常用一小块电池驱动，当电池失效时，相关配置和时间等都将丢失
三大核心硬件
CPU、内存（主存）、硬盘

  • 速度：硬盘 < 内存 < CPU。硬盘读写速度很慢，因此需要内存从硬盘快速提取数据（加载），交给 CPU 运行，可以通过高速缓存、寄存器提升运行速度

  • 性能：内存限制计算机的性能，可以通过加内存条提升性能，通过更换固态硬盘提升软件启动速度

3. 多线程与多核

  1. 单核：一个大脑
  2. 双核：一个大脑，两个半球
  3. 双核四线程：一个大脑，两个半球，每个半球一心两用
    • 双核 < 双核四线程 < 四核，即双核四线程为真双核，假四核

4. 指令集

  • 精简指令集：单条指令比较简单，单条指令能做的事较少，因此基于精简指令集写程序需要指令条数更多，但稳定性更高（ARM 芯片）

  • 复杂指令集：单条指令比较复杂，单条指令能做的事较多，因此基于复杂指令集写程序需要指令条数更少，但稳定性相对没有精简指令集高（x86）

5. 32位与64位

32位
  • CPU 一次性从内存取 32 个 2 进制数
64位
  • CPU 一次性从内存取 64 个 2 进制数

6. 磁盘

  • 数据都存放于一段一段的扇区，即磁道这个圆圈的一小段圆圈，从磁盘读取一段数据需要经历寻道时间和延迟时间

    1. 平均寻道时间：机械手臂从一个柱面随机移动到相邻的柱面的时间成为寻到时间，找到了磁道就以为着招到了数据所在的那个圈圈，但是还不知道数据具体这个圆圈的具体位置
    2. 机械臂到达正确的磁道之后还必须等待旋转到数据所在的扇区下，这段时间成为延迟时间

  • 机械磁盘找到数据需要经历的时间=平均延迟时间+平均寻道时间

    • 以 7200 转/min 的机械硬盘为例，

      ​ 转一圈：8ms
      ​ 平均延迟时间：转半圈 -> 4ms
      ​ 平均寻道时间：5ms

  • 所以机械磁盘慢是慢在找数据上，至于找到数据之后往内存读，这个速度是很快额

7. 计算机三层体系结构

  • 应用程序（以图形表现） –> （内核、操作系统接口层）–> 操作系统（即是代码） –> 计算机硬件

    1. 应用程序：对操作系统进行封装，精简操作，以此来控制操作系统接口
    2. 内核、操作系统接口：对操作系统进行分类、封装，简化操作
    3. 操作系统：对计算机硬件功能进行封装，然后通过直接控制硬盘，来间接控制其他硬件，即控制计算机运行
    4. 运行顺序：硬盘 –> 操作系统代码 –> 操作系统接口 –> 内核 –> 图形界面程序 –> 应用程序
    5. 操作顺序：应用程序（快捷方式） –> 图形界面程序（寻找路径） –> 操作系统接口 –> 内核 –> 硬盘 –> 内存 –> CPU

二、Python入门

  • 程序 = 数据 + 功能（代码）

1. 操作系统的安装与启动

  1. BIOS（基础操作系统）：Basic input output system

    • BIOS 存储与内存中，为 ROM 数据

    • BIOS 所配置的数据存储于内存中，为 CMOS 数据

  2. windows，Linux，Ios（主操作系统）

    • 存储于本地硬盘
    • 主系统配置数据存放于内存，为 RAM 数据

2. 平台

  • 平台 = 计算机硬件 + 操作系统（一种计算机 + 一种操作系统 即为 一种平台）

  • 跨平台性：开发出的软件可以在任意平台正常运行

3. 编程语言分类

  1. 低级语言（底层语言）：机器语言、汇编语言

    • 机器语言：直接用计算机能理解的二进制指令去编写程序，是直接在控制计算机硬件
      • 优点：运行效率高
      • 缺点：开发效率低
        • 开发一个简单小功能需要用到的二进制指令条数都非常多
        • 二进制指令非常难以记忆
    • 汇编语言：用英文标签取代二进制指令去编写程序，是直接在控制计算机硬件（只解决了机器语言难记的问题）

  2. 高级语言（上层语言）：用人类能理解的表达方式去编写程序[编译型（GO，C）、解释型（Python，Shell）] ，与操作系统进行工作

    • 编译型：编译型语言编写程序如果编译成功，第二次运行时不需要编译器

      源代码 —— 编译器 —— 计算机能理解的指令

    • 解释型：每次运行程序都要翻译，且解释型语言编写程序每次运行离不开解释器

    • 第一次运行 Python 程序，Python 解释器会生成字节码文件；第二次运行 Python 程序的时候，Python解释器运行的是字节码文件

      源代码 —— 解释器 ——计算机能理解的指令

      • 高级语言的开发效率受网速限制，且取决于语言的本身特性（网速 < 硬盘 < 内存 < CPU）
      • C语言保存了调用操作系统原生接口的功能，在高级语言中相对底层，所有语言的老父亲
      • Python语言三大特性：
        • 语法简洁，容易上手
        • 封装程度高
        • 非常丰富的第三方库
      • 机器语言 > 汇编语言 > 编译型 > 解释型（只比较语言，意义不大）
      • 开发效率和语言本身特性有关
